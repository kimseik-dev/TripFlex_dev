# 🕵️‍♀️ TripFlex Google Vision 403 에러 해결 가이드

대표님! 구글 경비원(403 Forbidden)을 뚫고 무사히 스캐너를 돌리기 위한 **3단계 긴급 체크리스트**예요. 🎨✨

---

## 1단계: Google Cloud Vision API 활성화 (가장 중요! 🗝️)

구글 클라우드에서 API만 생성한다고 바로 쓸 수 있는 건 아니에요! 이 기능을 **"사용 설정"** 해야 한답니다.

1.  **[Google Cloud Console](https://console.cloud.google.com/apis/library/vision.googleapis.com)** 에 접속하세요.
2.  사용 중인 **프로젝트**가 맞는지 상단에서 확인하세요.
3.  화면 중앙에 있는 **[사용(ENABLE)]** 버튼을 클릭하세요. (이미 활성화되어 있다면 '관리'라고 뜰 거예요!)

---

## 2단계: 결제 계정(Billing) 연결 확인 💸

무료 티어(월 1,000건)라도, 구글은 **"카드 등록이 된 결제 계정"**이 프로젝트에 연결되어 있어야 API 호출을 허용해줘요.

1.  **[결제(Billing)]** 메뉴로 이동하세요.
2.  현재 프로젝트에 유효한 결제 계정(신용카드 등)이 연결되어 있는지 확인하세요.
3.  만약 끊겨 있다면 결제 계정을 등록하고 프로젝트에 연결해 주세요!

---

## 3단계: API 키 제한 사항(Restrictions) 점검 🔒

API 키를 만들 때 보안을 위해 특정 도메인이나 IP만 허용하도록 제한을 걸었을 수도 있어요.

1.  **[사용자 인증 정보(Credentials)]** 메뉴로 이동하세요.
2.  사용 중인 API 키를 클릭합니다.
3.  **API 제한사항**: '제한사항 없음'이거나, 'Cloud Vision API'가 포함되어 있는지 보세요.
4.  **애플리케이션 제한사항**: 현재 로컬 개발 환경(`localhost`)에서도 호출할 수 있도록 되어 있는지 확인하세요. (일단 테스트를 위해 '없음'으로 두고 해보시는 걸 추천해요!)

---

## 🛠️ 영자가 도와주는 디버깅 (v310)

제가 방금 서버(`api/relay/route.ts`)에 **디버그 로그**를 한 줄 심어놨어요! ✨
터미널 로그에 아래와 같이 나오는지 확인해 보세요:

```text
[API_RELAY] API KEY LOADED: (첫5글자)*****... 🔑
```

- 만약 아예 비어있거나 `your-` 등으로 시작한다면, `.env` 파일의 키가 코드에 제대로 전달 안 된 거예요!
- `.env` 파일을 수정했다면 **서버를 껐다가 다시 켜야(Restart)** 적용된답니다! 🌈
